# Backend Engineer Reboot

A long-term self-study repository where I go beyond day-to-day ticket work and build deep, systematic understanding of backend engineering.

After 6+ years of professional experience with Java and Spring Boot — most recently building production systems at Rakuten Japan — I started this repo to take full ownership of the concepts I work with every day.

This isn't exam prep. It's engineering discipline.

---

## Philosophy

Every topic follows the same rule:

> **Note → Code → Commit. A topic isn't done until all three exist.**

Inspired by the Feynman Technique — if I can't write it clearly and implement it, I don't actually know it.

## Topics

**Java & JVM** — Java Core, Collections, Stream API, Functional Interfaces, Modern Java (17→21), JVM Memory, Concurrency

**Spring Ecosystem** — Spring Core (IoC/DI/AOP), Spring MVC, Spring Data JPA, Spring Security, JWT, OAuth2

**Data Layer** — SQL (Indexing, ACID, Transactions), Redis (Cache, Rate Limiting, Redlock), Elasticsearch

**Distributed Systems** — Microservices Patterns, Circuit Breaker, Saga Pattern, CQRS, API Gateway, Service Decomposition

**Messaging** — Kafka (Topics, Partitions, Ordering, DLQ, Outbox Pattern), RabbitMQ, ActiveMQ, Event-Driven Architecture

**DevOps & Observability** — Docker, Kubernetes, CI/CD (Jenkins), ELK Stack, Prometheus, Grafana, Distributed Tracing

**System Design** — Scalability, Load Balancing, CAP Theorem, Caching Strategies, Database Sharding
